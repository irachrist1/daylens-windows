# Issues

## Current Known Constraints

- Real-machine Windows validation still matters even when tests pass.
- Unsigned Windows builds will still trigger SmartScreen warnings.
- The repo is still historically named `daylens-windows` even though the app now spans macOS, Windows, and Linux.
- Linux still needs real-machine validation on both X11 and Wayland sessions even after CI packaging lands.

The later recap follow-on pass on 2026-04-19 added a deterministic daily / weekly / monthly recap surface inside `src/renderer/views/Insights.tsx`, backed by a new `src/renderer/lib/recap.ts` aggregator instead of provider-only AI calls. The new empty-state AI surface now opens with a Wrapped-inspired but evidence-grounded recap card: period tabs for day / week / month, tracked/focus/deep-stretch/context-switch metrics, top workstreams, standout artifacts, same-period comparison copy, and direct chat prompts that stay in the existing AI surface rather than becoming a new route.

A follow-on recap polish pass on 2026-04-19 restructured the recap around paced `RecapChapter` story beats (`headline` / `focus` / `artifacts` / `rhythm` / `change`) rendered as a numbered, dividered sequence in the hero card instead of a single-paragraph summary. The aggregator now surfaces deeper signals grounded in tracked data: the most switch-heavy block as an explicit "where focus broke" callout, the peak weekday and quiet-day count for week/month rhythm, an attribution `RecapCoverage` ratio with a visible partial-coverage ribbon when more than ~20% of tracked time sits in unnamed blocks, and dynamic prompt chips that reference the user's own top workstream and standout artifact instead of only generic templates. A later follow-up in the same pass tightened truthfulness at the edges: month-to-month comparisons now use symmetric matched windows on 31-vs-30-day boundaries with copy that explicitly says when only the first N days are being compared, and the top-workstreams list now keeps dominant unnamed work visible instead of overstating attribution coverage. Copy was rewritten to stay honest about thin data (short day, all-unnamed blocks, absent comparison period) and to avoid vanity framing. `tests/recap.test.ts` now covers chapter pacing, coverage honesty, weekly rhythm signals, and the matched-month window edge case (9/9 pass locally).

The cross-platform parity follow-on pass on 2026-04-19 closed several shared truthfulness gaps without overstating runtime proof. `src/shared/platformExpectations.ts` now centralizes platform-specific Settings copy for launch-on-login, quick access, and install/update expectations so macOS menu bar language is no longer treated as the default desktop story. The Tracking section now explains menu bar vs. Windows tray vs. Linux tray/AppIndicator behavior honestly, and the Updates section now explains DMG / Setup.exe / AppImage-DEB-RPM expectations more explicitly, including SmartScreen truthfulness on Windows and tray-shell caveats on Linux. `src/main/services/workspaceLinker.ts` now falls back to `This device` instead of the Windows-only `This PC` label when no hostname is available, and `src/main/services/distractionAlerter.ts` now actually respects the `distractionAlertsEnabled` setting instead of firing alerts even when the toggle is off. The Linux Notifications section also now warns when the current session does not expose native notification support.

The web-parity follow-on pass later on 2026-04-19 moved synced snapshots onto a real Snapshot v2 contract instead of leaving the web stuck on the older app-centric shape. `src/shared/snapshot.ts` now defines the additive v2 schema and `src/main/services/snapshotExporter.ts` exports Linux-safe `schemaVersion: 2` payloads with `focusScoreV2`, work-block summaries, recap payloads, coverage, top workstreams, standout artifacts, and routed entities while preserving the legacy fields for backward compatibility. The sibling `daylens-web` repo now accepts both snapshot versions, no longer rejects Linux devices at the validator/device/http layers, merges v2 snapshots in Convex, server-filters hidden apps/domains before the browser receives them, grounds web AI prompts on work blocks + recap + entities instead of only app tallies, and adds recap/work-block surfaces on the dashboard, history, chat, and a dedicated web recap route.
